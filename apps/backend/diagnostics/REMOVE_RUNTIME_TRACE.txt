# Instructions to Remove RUNTIME_TRACE Instrumentation

## Overview
All RUNTIME_TRACE diagnostic logs can be safely removed by searching for "RUNTIME_TRACE" and deleting those lines.

## Files Modified
The following files contain RUNTIME_TRACE instrumentation:

1. apps/backend/src/controllers/AuthController.ts
2. apps/backend/src/services/AuthService.ts
3. apps/backend/src/middleware/auth.ts
4. apps/backend/src/middleware/tenant.ts

## Removal Steps

### Option 1: Manual Removal
1. Open each file listed above
2. Search for "RUNTIME_TRACE"
3. Delete lines containing logger.info('RUNTIME_TRACE ...')
4. Delete lines containing logger.error('RUNTIME_TRACE ...')
5. Save files
6. Recompile TypeScript: npm run build
7. Restart server

### Option 2: Automated Removal (Linux/Mac)
```bash
cd apps/backend/src

# Remove RUNTIME_TRACE from controllers
sed -i '/RUNTIME_TRACE/d' controllers/AuthController.ts

# Remove RUNTIME_TRACE from services
sed -i '/RUNTIME_TRACE/d' services/AuthService.ts

# Remove RUNTIME_TRACE from middleware
sed -i '/RUNTIME_TRACE/d' middleware/auth.ts
sed -i '/RUNTIME_TRACE/d' middleware/tenant.ts

# Rebuild
npm run build
```

### Option 3: Automated Removal (Windows PowerShell)
```powershell
cd apps\backend\src

# Remove RUNTIME_TRACE from all files
Get-ChildItem -Recurse -Include *.ts | ForEach-Object {
    (Get-Content $_.FullName) | Where-Object { $_ -notmatch 'RUNTIME_TRACE' } | Set-Content $_.FullName
}

# Rebuild
npm run build
```

## Verification
After removal:
1. Search entire codebase for "RUNTIME_TRACE" - should return 0 results
2. Run TypeScript compiler: npm run build
3. Check for compilation errors
4. Start server and verify auth still works

1. Use git to revert changes: git checkout HEAD -- apps/backend/src/
2. Or re-apply the instrumentation manually
